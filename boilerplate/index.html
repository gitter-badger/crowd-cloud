Crowd Cloud Boilerplate

All content in a crowd cloud project can be either refering to http/https served content or to crowd-links.

The crowd-links have the following scheme:
crowd://discovery-server.tld/projectname/subfolder/file.png

If the discovery-server gets blocked, you might want to offer an alternative discovery-server ("network"). You can do so by offering an data-src attribute containing comma-seperated crowd-links: 

			<img src="crowd://blocked-server.org/projectname/image.png" data-src="crowd://alternative.com/projectname/image.png,crowd://alternative.org/projectname/image.png">

What kind of content is supported to be delivered via the crowd websocket protocol?

Currently, there are three types of content (in this early prototype phase): png images, stylesheets and javascript. Further content will be supported as the project improves over time.
